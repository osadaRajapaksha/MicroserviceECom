# Custom Behaviors

- **Version Control**: Always commit and push the code to GitHub at the end of each prompt/turn after making any changes.
- **Granular Commits**: To reach the target of 100-120 commits, you must build the project piece by piece and commit EXTREMELY granularly. Commit class-by-class, configuration-by-configuration, or component-by-component rather than making large feature-level commits.
- **Git Branching**: Use Git branching when developing new features or making significant changes, and merge them into the main branch when necessary.
